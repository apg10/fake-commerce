# 10-AGENT-COMMUNICATION-PROTOCOL.md
## Final Report Format
1. Files created or updated
2. Brief summary of each file
3. Anything intentionally not done
4. Whether any tests/commands were run
5. Recommended next task

## Review Pattern
- Only work on files listed as allowed
- Create worker safe output example files that match the current safe task
- Include the local model in the final report for all safe local files worked on when requested explicitly
- Stop after the current task scope is complete and report honestly
- Never claim validation unless a relevant test ran and passed
- Never touch files outside the allowed scope unless explicitly instructed
- Use the safe local model task queue unless the task is to create all files
